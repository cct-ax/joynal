# セキュリティポリシー

Joynal の脆弱性報告に関する方針です。

## 対応バージョン

セキュリティ修正の対象は最新の `main` ブランチ（本番デプロイ中のコード）のみです。過去のコミットや古いブランチへの個別対応は行いません。

## 脆弱性の報告方法

**脆弱性は公開 Issue や PR に書かないでください。** 修正前に内容が公開されると悪用される恐れがあります。

報告は以下のいずれかでお願いします。

1. **GitHub の Private vulnerability reporting（推奨）**
   リポジトリの **Security** タブ →「**Report a vulnerability**」から非公開で報告できます。
2. 上記が使えない場合は、**PL に直接連絡**してください。

報告には、再現手順・影響範囲・想定される攻撃シナリオなど、確認に必要な情報をできるだけ含めてください。

## 対応の流れ

1. 報告を受領し、内容を確認します。
2. 再現性・影響を評価します。
3. 修正を行い、必要に応じて関連する依存関係（Dependabot / CodeQL のアラート）もあわせて対応します。
4. 対応状況を報告者に連絡します。
