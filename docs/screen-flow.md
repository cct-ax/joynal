## 画面遷移図

```mermaid
flowchart TD
    Start([アクセス]) --> Auth{ログイン済み？}
    Auth -- No --> Login[/ログイン画面/]
    Auth -- Yes --> Role{ロール判定}

    Login -- ログイン成功 --> Role
    Login -- パスワードを忘れた --> Reset[/パスワードリセット画面/]
    Reset -- 確認コード送信 --> Reset
    Reset -- コード検証＋更新＋全セッション失効 --> Login

    Role -- 全員 --> Report[/日報画面/]
    Report -- 管理者のみ --> Admin[/管理画面/]

    Report -- ログアウト --> Login
    Admin -- ログアウト --> Login
```

## 各画面の役割（遷移上の位置づけ）

各画面の **UI 詳細仕様**（部品・バリデーション・レスポンシブ）は [画面 UI 仕様](./design-spec.md) を正典とする。本書では遷移の文脈で各画面が何のための場所かだけを示す。

| 画面 | パス | 遷移上の位置づけ |
|------|------|------|
| ログイン画面 | `/login` | 全ルートの入口。未ログインのアクセスはここへ集約され、成功で `/report` へ進む |
| パスワードリセット画面 | `/reset-password` | コード送信〜新パスワード設定を1画面で完結。更新成功で全セッション失効 → `/login?reset=success` へ戻る |
| 認証コールバック画面 | `/confirm` | メールリンクの着地点。トークン交換を自動処理して遷移するだけで固有 UI を持たない |
| 日報画面 | `/report` | ログイン後の全ロール共通ハブ。担当新人セレクタ・週ナビ・日報リスト・コメント・週次サマリーをロールに応じて出し分ける |
| 管理画面 | `/admin` | 管理者専用。ユーザー管理とメンター割り当てをタブで切り替える。非管理者は `auth.global.ts` で `/report` へ弾かれる |
