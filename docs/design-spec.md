# デザイン仕様書

> この仕様書はFigmaでのデザイン作成（Claude Design等）のインプットとして使用します。

---

## アプリ概要

**アプリ名**: jornal  
**用途**: 新人研修の日報管理Webアプリ  
**対象デバイス**: PC・タブレット・スマホ（レスポンシブ対応）  
**言語**: 日本語

---

## デザイントーン

- シンプル・クリーン
- 業務用途のため過度な装飾は不要
- 視認性重視（テーブル・フォームが多い）

---

## レスポンシブ方針

| ブレークポイント | 対象 |
|----------------|------|
| 1024px以上 | PC |
| 768px〜1023px | タブレット |
| 767px以下 | スマホ |

**スマホ時のレイアウト変更**:

- 日報画面（メンター・OJT用）: 2カラム → 1カラム（日報一覧 → タップで詳細に遷移）
- 管理画面: テーブルは横スクロール対応
- ヘッダー: ユーザー名を省略し、アイコンメニューに変更

## 画面仕様

---

### 1. ログイン画面

**概要**: 全ユーザーが使うログインページ。画面中央にカードを配置。

**コンポーネント**:

- アプリ名「jornal」（ページ上部中央、大きめのテキスト）
- メールアドレス入力フィールド（ラベル付き）
- パスワード入力フィールド（ラベル付き）
- ログインボタン（Primary、幅100%）
- 「パスワードをお忘れの方はこちら」テキストリンク（中央寄せ）

**バリデーションエラー表示箇所**:

- 各フィールドの下にエラーメッセージ（赤文字）

---

### 2. パスワードリセット画面

**概要**: メールアドレスを入力してリセットメールを送信する画面。

**コンポーネント**:

- タイトル「パスワードリセット」
- 説明文「登録済みのメールアドレスを入力してください。」
- メールアドレス入力フィールド
- 「リセットメールを送信」ボタン（Primary、幅100%）
- 「← ログイン画面に戻る」テキストリンク

---

### 3. 日報画面（新人用）

**概要**: 新人が日報を週単位で確認・入力する画面。1週間分（月〜金）の日報を曜日付きで表示し、その週のメンター・OJTコメントもあわせて確認できる。

**ヘッダー**:

- 左: アプリ名「jornal」
- 右: ログインユーザー名 ＋ 「ログアウト」リンク

**検索エリア**:

- キーワード入力フィールド（やったことの全文検索）
- 期間指定（From〜To 日付入力）
- 「検索」ボタン
- 検索結果は週単位の表示を崩さず、ヒットした日報をハイライト表示する

**週ナビゲーション**:

- 「< 前の週」ボタン ／ 「次の週 >」ボタン
- 現在の週を中央に表示（例: 2026/04/20（月）〜 04/24（金））
- 「今週」ボタンで今週に戻る

**週間日報リスト**:

- 月〜金を1行ずつ表示（日報が未入力の日も行を表示）
- 各行の表示内容:
  - 日付と曜日（例: 04/21（月））
  - 出勤〜退勤時間（未入力の場合は「--」）
  - やったことの先頭100文字（省略表示、長文対応）
  - 入力済みの場合:「編集」ボタン → 編集モーダルを開く
  - 未入力の場合:「入力」ボタン → 入力モーダルを開く

**週次コメントエリア**（週間日報リストの下部）:

- 「メンターコメント」ラベル ＋ メンターの名前
  - コメント本文（長文対応、折り返し表示）
  - 未入力の場合「コメントはまだありません」
- 「OJTコメント」ラベル ＋ OJTの名前
  - コメント本文（長文対応、折り返し表示）
  - 未入力の場合「コメントはまだありません」

**日報入力・編集モーダル**:

- タイトル: 「日報を入力」または「日報を編集」
- 日付（表示のみ、変更不可）
- 曜日（表示のみ）
- 出勤時間（Time Picker、必須）
- 退勤時間（Time Picker、必須）
- やったこと（テキストエリア、複数行・縦に拡張、必須）
- 「キャンセル」ボタン（Secondary）
- 「保存」ボタン（Primary）
- スマホでは全画面モーダル

---

### 4. 日報画面（メンター・OJT用）

**概要**: 担当新人の日報を週単位で閲覧し、週次コメントを入力する画面。

**ヘッダー**:

- 左: アプリ名「jornal」
- 右: ログインユーザー名 ＋ 「ログアウト」リンク

**担当新人セレクター**:

- 「担当新人:」ラベル ＋ ドロップダウン（複数担当がいる場合に切り替え）

**検索エリア**:

- キーワード入力フィールド（やったことの全文検索）
- 期間指定（From〜To 日付入力）
- 「検索」ボタン
- ヒットした日報をハイライト表示

**週ナビゲーション**:

- 「< 前の週」ボタン ／ 「次の週 >」ボタン
- 現在の週を中央に表示（例: 2026/04/20（月）〜 04/24（金））
- 「今週」ボタンで今週に戻る

**週間日報リスト**:

- 月〜金を1行ずつ表示（日報が未入力の日も行を表示）
- 各行の表示内容:
  - 日付と曜日（例: 04/21（月））
  - 出勤〜退勤時間（未入力の場合は「--」）
  - やったことの先頭100文字（省略表示、長文対応）
  - 「詳細」ボタン → 詳細モーダルを開く（やったことの全文を表示）

**週次コメント入力エリア**（週間日報リストの下部）:

- 「この週のコメント」タイトル ＋ 対象週の表示（例: 04/20 〜 04/24）
- コメント入力テキストエリア（複数行・縦に拡張、長文対応）
- 「送信」ボタン（Primary、右寄せ）
- すでにコメント済みの場合: 入力済みコメントを表示し「編集」ボタンを表示

**日報詳細モーダル**:

- 日付・曜日
- 出勤時間・退勤時間
- やったこと（全文表示、長文対応・縦スクロール）
- 「閉じる」ボタン

---

### 5. 管理画面

**概要**: 管理者専用。タブで3機能を切り替える。

**ヘッダー**:

- 左: アプリ名「jornal」
- 右: ログインユーザー名 ＋ 「ログアウト」リンク

**タブ**: 「ユーザー管理」「メンター割り当て」「全日報一覧」

---

#### タブ①: ユーザー管理

- 「＋ ユーザーを追加」ボタン（右上）
  - クリックでモーダル表示
  - モーダル内容: 名前・メール・役割のフォーム ＋ 招待ボタン

- ユーザー一覧テーブル:
  - カラム: 名前 / メール / 役割（ドロップダウン）/ 操作
  - 役割ドロップダウン: 新人 / メンター / OJT / 管理者
  - 操作列: 「無効化」ボタン（Danger）

---

#### タブ②: メンター割り当て

- 「年度:」ドロップダウン（例: 2026）

- 割り当てテーブル:
  - カラム: 新人 / メンター（ドロップダウン）/ OJT（ドロップダウン）
  - ドロップダウンの選択肢はメンター・OJTロールのユーザー一覧

- 「保存」ボタン（Primary、右下）

---

#### タブ③: 全日報一覧

- フィルターエリア:
  - 新人フィルター: ドロップダウン（全員 or 個別選択）
  - 期間: From日付 〜 To日付
  - キーワード: テキスト入力 ＋ 「検索」ボタン

- 日報一覧テーブル:
  - カラム: 新人名 / 日付 / 出勤 / 退勤 / やったこと（省略表示）
  - 日付降順で表示

---

## 共通コンポーネント

| コンポーネント | 仕様 |
|--------------|------|
| ヘッダー | 全ページ共通。左: アプリ名、右: ユーザー名＋ログアウト |
| Primary ボタン | メインアクション用 |
| Secondary ボタン | キャンセルなど補助アクション用 |
| Danger ボタン | 削除・無効化など破壊的操作用（赤系） |
| テキストリンク | ページ遷移用 |
| 日報入力モーダル | 日報の入力・編集に使用。スマホでは全画面 |
| 日報詳細モーダル | やったことの全文表示用（長文対応・スクロール） |
| ユーザー追加モーダル | 管理者のユーザー招待フォーム用 |
| テキストエリア | やったこと・コメントなど長文入力に使用。入力量に応じて縦に拡張する |
| トースト通知 | 送信成功・失敗のフィードバック |
| エラーメッセージ | フィールド直下に赤文字で表示 |
