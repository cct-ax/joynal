# デザイン仕様書

> この仕様書はFigmaでのデザイン作成（Claude Design等）のインプットとして使用します。

---

## アプリ概要

**アプリ名**: Joynal  
**用途**: 新人研修の日報管理Webアプリ  
**対象デバイス**: PC・タブレット・スマホ（レスポンシブ対応）  
**言語**: 日本語

---

## デザイントーン

- シンプル・クリーン
- 業務用途のため過度な装飾は不要
- 視認性重視（テーブル・フォームが多い）

## 画面仕様

---

### 1. ログイン画面

**概要**: 全ユーザーが使うログインページ。画面中央にカードを配置。

**コンポーネント**:

- アプリ名「Joynal」（ページ上部中央、大きめのテキスト）
- メールアドレス入力フィールド（ラベル付き）
- パスワード入力フィールド（ラベル付き）
- ログインボタン（Primary、幅100%）
- 「パスワードをお忘れの方はこちら」テキストリンク（中央寄せ）

**バリデーションエラー表示箇所**:

- 各フィールドの下にエラーメッセージ（赤文字）

---

### 2. パスワードリセット画面

**概要**: メールアドレスを入力してリセットメールを送信する画面。

**コンポーネント**:

- タイトル「パスワードリセット」
- 説明文「登録済みのメールアドレスを入力してください。」
- メールアドレス入力フィールド
- 「リセットメールを送信」ボタン（Primary、幅100%）
- 「← ログイン画面に戻る」テキストリンク

---

### 3. 日報画面（新人・メンター・OJT共通）`/report`

**概要**: 週単位で日報とコメントを確認できる共通画面。レイアウトは全ロール同一で、ロールによって操作できる内容が異なる。

**ヘッダー**:

- 左: アプリ名「Joynal」
- 右: ログインユーザー名（クリックでドロップダウン表示）
  - 「パスワード変更」→ パスワード変更モーダルを開く
  - 「ログアウト」

**新人セレクター**（メンター・OJT・管理者のみ表示）:

- 「対象:」ラベル ＋ ドロップダウン
  - **メンター・OJT**: 担当新人のみ選択肢に表示
  - **管理者**: 全新人が選択肢に表示
- **管理者の初期状態**: 未選択（空白）。画面中央に「表示したい新人の日報を選んでください」と表示し、週間リストは非表示

**週ナビゲーション**:

- 「< 前の週」ボタン ／ 「次の週 >」ボタン
- 現在の週を中央に表示（例: 2026/04/20（月）〜 04/24（金））
- 「今週」ボタンで今週に戻る
- 週ジャンプ: 日付ピッカーで任意の週に直接移動できる（1年分を効率よく閲覧するため）

**週間日報リスト**:

- 月〜金を1行ずつ表示（日報が未入力の日も行を表示）
- 各行の表示内容:
  - 日付と曜日（例: 04/21（月））
  - 出勤〜退勤時間（未入力の場合は「--」）
  - やったことの先頭80文字（省略表示、長文対応）
  - 気分（★星評価、mood が入力済みの場合のみ表示。未入力の場合は非表示）
- ロール別の操作:
  - **新人**: 未入力の行→「入力」ボタン、入力済みの行→「編集」ボタン（モーダルを開く）。未入力の行は出勤・退勤・やったことは空白表示
  - **メンター・OJT・管理者**: 入力済みの行→「詳細」ボタン（インライン展開で全文表示）。未入力の行は空白表示（ボタンなし）

**週次コメントエリア**（週間日報リストの下部）:

- PC: 2カラムグリッド（メンター ／ OJT 横並び）、スマホ: 1カラム縦積み
- 各カラム内:
  - ロールバッジ（例: メンター） ＋ 担当者名
  - コメント本文（長文対応、折り返し表示）
  - 未入力の場合「コメントはまだありません」（イタリック・薄色）
  - 自分のロールに一致する場合のみ「入力」または「編集」ボタンを表示 → コメント入力モーダルを開く
- コメントは**1週1コメントの上書き仕様**

**週次サマリーエリア**（週間日報リストの下部。mood グラフは mentor/ojt/admin のみ、AI サマリーは新人本人も対象）:

- カード見出し「週次サマリー」。対象は担当新人セレクタで選択中の新人
- **気分の推移グラフ**（直近8週・日次）:
  - `@unovis/vue` の SVG ライン（`MoodTrendChart.client.vue`・クライアント専用描画）
  - y 軸 1〜5 固定、x 軸は日付（M/DD・週単位の目盛）、線・点は semantic primary 色
  - mood 未入力日は線のギャップ（点を打たない）
  - 取得中はスケルトン、データ0件は EmptyState「この期間に記録された気分がありません」
- **AI 週次サマリー**（`AiSummaryPanel`）: メンター/OJT/管理者は観察視点、新人本人は自分の振り返り（audience をサーバー導出）。生成/再生成ボタン、鮮度バッジ（日報更新時「内容が更新されています」）、「参考情報」注記つき。評価・断定・代筆はしない

**日報入力・編集モーダル**（新人のみ）:

- タイトル: 「日報を入力」または「日報を編集」
- 日付（表示のみ、変更不可）
- 曜日（表示のみ）
- 出勤時間（Time Picker、必須）
- 退勤時間（Time Picker、必須）
- やったこと（テキストエリア、複数行・縦に拡張、必須）
- 「ヒントを見る（AI）」リンクボタン（任意）: 入力中のドラフト＋気分から振り返りの深掘り質問2〜3個＋短評を `CoachHints` に表示。**本文への挿入・コピー導線は置かない**（代筆防止）。ローディング中はボタンに loading、失敗時はトースト
- 気分（星5つで評価、任意）
  - ★☆☆☆☆〜★★★★★ の5段階
  - クリックで選択、再クリックで選択解除（未入力に戻る）
  - 選択中の星とそれ以下をハイライト表示（通常の星マーク評価UI）
- 「キャンセル」ボタン（Secondary）
- 「削除」ボタン（Danger、編集モード時のみ表示）→ 確認ダイアログを挟んでから削除
- 「保存」ボタン（Primary）
- スマホでは全画面モーダル

**日報詳細（インライン展開）**（メンター・OJT・管理者）:

- 出勤時間・退勤時間
- 気分（星★で表示、入力がない場合は非表示）
- やったこと（全文表示、長文対応・縦スクロール）
- 再クリックで折りたたむ（▲/▼ ボタン）

**コメント入力モーダル**（メンター・OJTのみ）:

- タイトル: 「週次コメントを入力」または「週次コメントを編集」
- 対象週の表示（例: 2026/04/20（月）〜 04/24（金））
- コメント入力テキストエリア（必須、縦に拡張）
- 「キャンセル」ボタン（Secondary）
- 「保存」ボタン（Primary）
- スマホでは全画面モーダル

---

### 4. 管理画面

**概要**: 管理者専用。タブで2機能を切り替える。

**ヘッダー**:

- 左: アプリ名「Joynal」
- 右: ログインユーザー名（クリックでドロップダウン表示）
  - 「パスワード変更」→ パスワード変更モーダルを開く
  - 「ログアウト」

**タブ**: 「ユーザー管理」「メンター割り当て」

---

#### タブ①: ユーザー管理

- 「＋ ユーザーを追加」ボタン（右上）
  - クリックでユーザーフォームモーダル（追加モード）を表示
  - モーダル入力項目: 社員ID（必須・手入力）・名前・メールアドレス・役割
  - ボタン: 「招待する」（Primary）
  - 招待すると初期パスワード設定用の確認コード（OTP）メールが届く

- ユーザー一覧テーブル:
  - カラム: 社員ID / 名前 / メール / 役割（バッジ表示） / 操作
  - 無効化されたユーザーは行を半透明（opacity 0.45）で表示
  - 操作列:
    - 「編集」ボタン（Outline）→ ユーザーフォームモーダル（編集モード）を開く（社員ID・名前・メール・役割を変更可）
    - 有効ユーザー: 「無効化」ボタン（Danger Outline）
    - 無効ユーザー: 「有効化」ボタン（Outline）

---

#### タブ②: メンター割り当て

- 割り当てテーブル:
  - カラム: 新人 / メンター（ドロップダウン）/ OJT（ドロップダウン）
  - ドロップダウンの選択肢はメンター・OJTロールのユーザー一覧

- 「保存」ボタン（Primary、右下）

> **注意**: メンター・OJTが途中で交代した場合も、新しい担当者は過去の日報を閲覧できる（RLSポリシーで`mentor_assignments`の全レコードを参照する設計とする）

## 共通コンポーネント

| コンポーネント | 仕様 |
|--------------|------|
| ヘッダー | 全ページ共通。左: アプリ名、右: ユーザー名（クリックでドロップダウン）＋ログアウト |
| Primary ボタン | メインアクション用 |
| Secondary ボタン | キャンセルなど補助アクション用 |
| Danger ボタン | 削除・無効化など破壊的操作用（赤系） |
| テキストリンク | ページ遷移用 |
| 日報入力モーダル | 日報の入力・編集に使用。スマホでは全画面 |
| 日報詳細（インライン展開） | やったことの全文表示。カードをクリックで展開・折りたたみ |
| コメント入力モーダル | 週次コメントの入力・編集。スマホでは全画面 |
| パスワード変更モーダル | ヘッダーのユーザー名クリック → ドロップダウン → 「パスワード変更」から開く。現在のパスワード・新しいパスワード・確認用パスワードを入力 |
| ユーザーフォームモーダル | 管理者のユーザー招待・編集フォーム用（追加/編集兼用・社員ID・名前・メール・役割） |
| テキストエリア | やったこと・コメントなど長文入力に使用。入力量に応じて縦に拡張する |
| トースト通知 | 送信成功・失敗のフィードバック |
| エラーメッセージ | フィールド直下に赤文字で表示 |
