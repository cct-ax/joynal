# 用語集

このページで分かること: アプリ全体で使う**役割名**と**ドメイン用語**の定義。各ドキュメントの表記はこの一覧に統一する。

> 関連: [要件定義](./requirements.md) ・ [DB 設計](./db-design.md) ・ [RLS 設計](./rls-design.md)

## 役割（ロール）

`profiles.role` の値。権限は Supabase の RLS ポリシーで強制し、フロントのガードはあくまで補助（詳細は [RLS 設計](./rls-design.md)）。

| 値 | 日本語 | 説明 |
|------|--------|------|
| `trainee` | 新人 | 日報を書く対象者。自分の日報のみ作成・編集できる。 |
| `mentor` | メンター | 担当新人の日報を閲覧し、週次コメントを書く。 |
| `ojt` | OJT | メンターと同様に担当新人を指導する役割。 |
| `admin` | 管理者 | ユーザー招待・無効化・メンター割り当てを行う。`/admin` にアクセスできる。 |

## ドメイン用語

| 用語 | 説明 |
|------|------|
| 日報（daily report） | 新人が1日ごとに記録する出退勤時刻・業務内容・気分。`daily_reports` テーブル。 |
| 気分（mood） | 日報に付ける 1〜5 の主観評価。週次の推移グラフに使う。 |
| 週次コメント | メンター/OJT が新人へ週単位で書くフィードバック。`comments` テーブル。 |
| メンター割り当て | 新人とメンター/OJT の年度ごとの対応関係。`mentor_assignments` テーブル。 |
| 週次サマリー（weekly summary） | その週の日報を AI が要約した参考テキスト。`ai_summaries` にキャッシュする。 |
| AI コーチング | 日報入力時に AI が出す深掘り質問＋短い励まし。本文の代筆はしない。 |
| audience | 週次サマリーの語り口。`self`（新人本人の振り返り）/ `mentor`（観察視点）。サーバーが導出する。 |
| 鮮度 | 週次サマリー生成後にその週の日報が更新されたか。再生成を促すバッジに使う。 |
| RLS（Row Level Security） | PostgreSQL の行レベル権限。誰がどの行を読み書きできるかを DB 側で強制する仕組み。 |
| JWT 転送 | フロントの認証 Cookie（JWT）を `serverSupabaseClient` がそのまま Supabase へ渡し、RLS をユーザー権限で効かせる方式。 |
